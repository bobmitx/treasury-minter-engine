            Proposal Submission

            Community members can submit logo proposals for any V1 through V4 minter token apart of the "System Tokens" box by providing:

            • Valid token contract address
            • High-quality logo image (PNG, JPG, GIF up to 5MB)
            • Can only upload Logo proposals through the front end
            • Make sure the proposed token is from the "System Tokens" box
            • Make Sure you have signed login message when first connecting to site for logo icon functionality

            Voting Process

            Once submitted, proposals enter the voting phase where:

            • Holders of 25 crow tokens can vote YES or NO on each proposal
            • Each wallet can vote once per proposal
            • If half of the total number wallets holding 25 crows tokens vote YES, then the proposal is approved
            • If Half of the total number of wallet holding 25 crows tokens vote NO, then the proposal is rejected
            • If there is a Tie betweem YES and NO votes, then the proposal is rejected
            • Vote counts are displayed in real-time
            • If a address is currentlty active and awaiting approval, that proposal has to end before that address can be submitted for approval again

            Approval & Implementation

            Successful proposals are moved to the "Approved" tab where:

            • Approved logos become the official token representation
            • The community can view all approved proposals
            • Proposals with sufficient votes are automatically approved

            How to Vote Manually

            • Call getActiveProposals() to get the current active proposals, the proposal ID's will be displayed in order
            • Call GetProposal(proposalID) to get the details of the proposal, can only view Logos on the front end
            • Call Vote(proposalID, true) to vote yes, or Vote(proposalID, false) to vote no

            Community Guidelines

            To ensure quality submissions:

            • Ensure images are clear and appropriately sized
            • Verify token addresses before submission
            • Vote responsibly on community proposals, votes are final and cannot be changed

            Frequently Asked Questions

            What is the Treasury System?
            The Treasury System is a decentralized ecosystem that allows users to create, mint, and manage various types of tokens on the blockchain. It involves multiple minter types (V1-V4) for minting, different token creation needs, and executing functions.
            What are the different minter types, and how do they work?
            • V1 - Tbill Minter: All tokens launched off the TBILL Minter are mintable 1:1 with treasury bill, any token launched off the V1 Tbill Minter will always have its parent as Treasury Bill.
            • V2 - Federal Minter: The V2 Federal Minter works a bit differenlty, V2 Tokens can only have Fed through BAR set as their parent token, its mintable 1:1 parent to child but has a claim function making a infinite mint exploit possible. With The V2 coins you have multiple functions like claim, publish, debenture, and more. If a token is published by a owneer then debenture will = false and that means you cannot use that token as a claim address to infinitely mint with. The V2 federal minter is meant for sandboxing and testing with future gaming functions on the later to be released Yue contract.
            • V3 - Index Minter: Tokens launched from the V3 Index Minter can actually have any token of their choosing as the Parent, there is a multiplier to mint for V3 tokens that increases every time 1,111,111,111 are minted.
            • V4 - Personal Minter: You can also choose any token to be the parent for V4 Personal Minter, there is also a multiplier factor for V4 tokens that increases every time the starting supply of the child token is minted.
            How do I Exploit/Infinite Mint V2 Federal Tokens/Spines?
            In the frontend you can infinitely mint Marias example V2 federal spine (FDIC - BAR), you can also infinitely mint custom V2 federal minter spines and tokens that are added into your spine manager.
          • How To Infinite Mint -- Choose a V2 Federal Minter token to mint, you use the parent to mint the child like normal except before minting you provide a claim address, when you go to mint the child with the parent you use your claim address token to claim back the parent you just used to mint your desired token, you claim the parent with the claim address token 1 for 1. This claim address must be a V2 Fedreal Minter token that has not been published yet, making debnture true. It will tell you if debenture is true or false when you input a claim address (green check for debenture = true, red x for debenture = false). If you do not enter a claim address then it will intiaite a normal mint without claiming.
            • How do I create a new token?
            Click the "Create Token" button, select your desired minter type, fill in the token details (name, symbol, initial mint amount), and click "Create Token". Make sure you have sufficient MV tokens (0xA1BEe1daE9Af77dAC73aA0459eD63b4D93fC6d29) for the initial mint. Initial supply is minted using MV tokens, 1 Mv token = 1 initial supply.
            How do I create a V2 Federal Spine?
            You first launch a V2 Federal Minter token with FED as the starting parent, from there launch a new V2 Federal token and set the parent contract address to the previous V2 Minter token you created, repeat this process up until the last token in your spine (you can choose how long your spine is, stop adding tokens to end the spine). You can view your spine by adding the contracts in order to your spine manager.
            How do I use the Federal Spine Manager?
            Press the plus symbol and import your custom V2 Federal Spine in order into the spine manager, You can also import V2 Federak Minter custom tokens not in a a particular spine to the spine manager. There is a infinite mint button for all V2 federal Minter tokens added into the spine manager for easy exploitation.
            How do I import existing tokens?
            Click the Import Token button, select the minter type and enter the token address you want to import. It will automatically fetch the parent token for you.
            Where can I see my imported tokens?
            You can see your imported tokens by clicking the Imported Tokens button in the top left corner of the page.
